# Gunners Pulse

A football management web app built with Django. It has two sides: one for the coach and one for the player.

---

## Overview

Gunners Pulse is a university graduation project. The idea is to bring all football-related data into one place — performance stats, match history, training sessions, tactics, injuries, and calendar events.

The coach can manage everything from one place. The player logs in and sees only their own data.

---

## Features

### Coach
- Dashboard — season overview, match schedule, squad stats
- Performance — player stats, tactical data, match history
- Player Comparison — side-by-side stats with radar chart
- Tactics — formation builder and role assignment
- Training — create and publish training sessions with drills
- Medical Center — log injuries and track recovery
- Calendar — manage season events

### Player
- Dashboard — personal stats, upcoming matches, coach notes
- Performance — match-by-match breakdown with charts
- Tactics — view assigned role in the team formation
- Training — see the latest published training session
- Calendar — personal view of season schedule

## Getting Started

**Requirements:** Python 3.10+

```bash
# Clone the repo
git clone https://github.com/your-username/gunners-pulse.git
cd gunners-pulse

# Activate virtual environment
source .venv/bin/activate        # macOS / Linux
.venv\Scripts\activate           # Windows

# Install dependencies
pip install -r requirements.txt

# Run migrations
python manage.py migrate

# Load data (run in this order)
python manage.py load_in_data
python manage.py load_in_matches
python manage.py load_player_dashboard
python manage.py load_Perf_players_data
python manage.py load_all_stats
python manage.py load_Perf3_tact_data
python manage.py load_Perf4_match_history_data
python manage.py load_player_performance
python manage.py load_players
python manage.py load_all_data
python manage.py load_player_tactics
python manage.py load_training_seed
python manage.py create_player_users
python manage.py load_med_players
python manage.py load_med_injury_meta
python manage.py load_calendar_data
python manage.py load_player_calendar

# Start the server
python manage.py runserver
```

Go to `http://127.0.0.1:8000/`

**Default player password:** `12345678` (generated by `create_player_users`)

## How Data Works

JSON files in `static/data/` are loaded into the database using management commands. Django views then query the database, pass the data to templates as JSON, and JavaScript renders the charts and tables.
